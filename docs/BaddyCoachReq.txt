This is a strong candidate for a production-grade badminton analytics platform because you already have three key research assets:

* TrackNetV3 (shuttle tracking)
* ShuttleSet (largest badminton analytics dataset)
* BST (state-of-the-art stroke classification transformer)
* MMAction2 skeleton-action ecosystem (ST-GCN, 2s-AGCN, PoseC3D)

The biggest mistake would be rebuilding these capabilities from scratch. The PRD below assumes **reuse-first architecture** with custom development focused on orchestration, analytics, coaching intelligence, player attribution, scalability, and reporting.

---

# Product Requirements Document (PRD)

# Product Name

**Badminton Post-Match Coaching Assistant (BMCA)**

Version: 1.0

Status: MVP + Production Roadmap

Owner: Product Management

Target Users:

* Competitive badminton players
* Academy coaches
* Parents of junior players
* Performance analysts
* Sports scientists

---

# 1. Problem Statement

Current badminton video analysis solutions provide:

* Shuttle tracking
* Stroke classification
* Rally segmentation

However they do not provide:

* Player-specific coaching analysis
* Technical feedback
* Tactical insights
* Fitness trends
* Long-match analytics
* Explainable coach recommendations

Coaches spend several hours manually reviewing match videos to identify:

* Shot selection issues
* Footwork inefficiencies
* Positioning errors
* Tactical patterns
* Fatigue indicators

The product should automatically analyze a recorded badminton match and generate coach-grade insights for a selected player.

---

# 2. Product Goal

Convert a raw badminton match video into:

* Rally timeline
* Stroke timeline
* Technical assessment
* Tactical assessment
* Fitness assessment
* Match summary
* Drill recommendations

without requiring manual tagging.

# 13. Explainability Requirements

Every recommendation must reference evidence.

Example:

```json
{
 "finding":
 "Late recovery after rear-court clears",

 "evidence":
 [
   "43 occurrences",
   "avg recovery 1.4 sec",
   "peer benchmark 0.9 sec"
 ]
}
```

No recommendation without supporting metrics.

## Recommended Implementation Order

1. Court calibration + homography
2. Player tracking + player selection
3. TrackNetV3 integration
4. Hit-frame localization engine
5. BST integration
6. Rally segmentation
7. Tactical analytics
8. Footwork analytics
9. Coach recommendation engine
10. Report UI

This sequence minimizes risk because **hit-frame localization and player attribution are the two components that most directly affect stroke-classification accuracy and coaching quality**.
