# Fix: BST Player Ordering and det_bbox_lookup

**Root causes identified:**
1. BST player ordering SWAPPED in colab (p0=near, p1=far) vs training (p0=far/top, p1=near/bottom)
2. det_bbox_lookup uses track_id iteration order instead of side assignment

---

## Fix 1: BST Player Ordering in Colab

**File:** `colab/pipeline.py` line 1296

**Current (WRONG):**
```python
for p_idx, pid in enumerate(['player_1', 'player_2']):
```
Where player_1=near, player_2=far. So p0=near(bottom), p1=far(top).

**Fix:**
```python
for p_idx, pid in enumerate(['player_2', 'player_1']):
```
Now p0=far(top)=Top_player, p1=near(bottom)=Bottom_player. Matches training convention.

---

## Fix 2: det_bbox_lookup in Colab

**File:** `colab/pipeline.py` lines 1612-1627

**Current (WRONG):**
```python
tid_to_pid = {}
for d in frame_dets:
    tid = d.get("track_id", 0)
    if tid not in tid_to_pid:
        tid_to_pid[tid] = f"player_{len(tid_to_pid)+1}"  # order depends on iteration
```

**Fix:** Build from side-assigned detections using the `side` field:
```python
# Use side-based assignment (matches pose data)
det_bbox_lookup = {}
for d in player_detections:
    side = d.get("side", "near")
    pid = "player_1" if side == "near" else "player_2"
    frame = d.get("frame")
    if frame is not None:
        if pid not in det_bbox_lookup:
            det_bbox_lookup[pid] = {}
        det_bbox_lookup[pid][frame] = d["bbox"]
```
